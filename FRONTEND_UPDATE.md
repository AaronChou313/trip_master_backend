  功能说明

  个人设置页面 (/profile)

  可修改的选项：
  1. 用户名 - 输入新的用户名
  2. 邮箱 - 输入新的邮箱地址
  3. 密码 - 可选修改密码
    - 需要输入当前密码
    - 需要输入新密码并确认

  其他功能：
  - 重置按钮 - 恢复为当前保存的值
  - 删除账户 - 危险区域，删除前需输入密码确认

  验证规则：
  - 用户名必填
  - 修改密码时必须提供当前密码
  - 新密码和确认密码必须一致
  - 新密码长度至少为6位
  - 如果不填新密码，则只修改用户名和邮箱

  ---
  后端需要的API
  ┌─────────────────────┬──────────────┬─────────────────────────────────────────────────────┬────────┐
  │        接口         │     方法     │                        说明                         │ 请求体 │
  ├─────────────────────┼──────────────┼─────────────────────────────────────────────────────┼────────┤
  │ PUT /api/auth/me    │ 更新用户信息 │ { username, email, currentPassword?, newPassword? } │        │
  ├─────────────────────┼──────────────┼─────────────────────────────────────────────────────┼────────┤
  │ DELETE /api/auth/me │ 删除用户账户 │ { password }                                        │        │
  └─────────────────────┴──────────────┴─────────────────────────────────────────────────────┴────────┘
  注意事项：
  - 所有接口都需要在 Authorization header 中携带 token
  - currentPassword 和 newPassword 是可选的，只有在修改密码时才需要
  - 删除账户需要密码验证
